# 运行节点

本指南旨在迅速介绍如何在 Fuel 区块链上运行本地节点的过程。

## 什么是节点？

在 Fuel 区块链的上下文中，节点通常被称为“客户端”，是一种软件，用于下载和维护 Fuel 区块链的副本。它验证每个区块和交易的真实性，确保您的副本始终保持最新并与网络同步。

有关最新版本的 Fuel 客户端，请访问[此处](https://github.com/FuelLabs/fuel-core)。

## 了解 Fuel 网络的测试网共识机制

Fuel 网络的测试网运行在权威证明（PoA）共识机制上。以下是简要概述：

**验证人**：在 PoA 中，有特定的实体，称为验证人或“权威”，负责创建新区块和验证交易。与其他共识机制（如工作量证明（PoW）或股权证明（PoS））不同，其中验证人是基于计算能力或股权选择的，PoA 验证人是基于他们在网络中的声誉和可信度选择的。

**PoA 的优势**：PoA 提供更快的交易时间，并且需要更少的计算能力，使其能源效率更高。网络的安全性和完整性由所选验证人的可信度维护。

## 为什么要运行自己的节点？

运行自己的节点有几个优点：

1. **查询自由**：通过托管自己的节点，您可以执行更多的查询而不会遇到任何速率限制。
2. **网络独立性**：拥有自己的节点可以确保您不依赖于第三方服务，使您完全控制与 Fuel 区块链的交互。

## 硬件要求

|  硬件      | 最低配置   | 推荐配置    |
|------------|------------|-------------|
|  处理器    |  2 核      |  8 核       |
|  内存      |  4 GB      |  12 GB      |
|  存储      |  30 GB     |  100 GB     |

对于低 API 流量，AWS m5.large 应该完全足够。但是，我们建议使用 AWS m5.4xlarge 实例，以匹配我们用于运行网络的配置。

> 对于部署简单合约和本地测试合约交互等常规任务，无需满足下面的所有硬件要求。
